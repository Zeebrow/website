things I want to remember:

  • everytime you make a new html element, make an associated css selector. Waiting until the dom is full of elements to make css is cumbersome?
  • consider wrapping "sections" of a webpage in their own container elements
  • consider setting a font-size on the html root, and using rem units to change font size in the document. seems like this would help improve "stylistic" consistency

new things that probably won't stick:

  • giving a contaning element padding: gives child elements something to push-off of, but changes the parent's dimensions. Use overflow: hidden; instead of padding:!
  • filter: blur(3px)
  • box-shadow: 0 0 3px 3px rgb()
  • border-radius: 9px;
  • transform: rotate(0.6deg);
  • * {box-sizing: border-box;} to "normalize"
  • use a meta tag with name=viewport content="width=device-width,initial-scale=1"
  • text-transform: uppercase;
  • object-fit: cover; to maintain image aspect ratio, preventing them from distoring
  • font-weight: 800;
  • letter-spacing: 0.15px;
  • :not pseudo selector
  • {?}link href="https://fonts.googleapis.com/css?family=Open+Sans:400,700,800" rel="stylesheet"{?} and font-family: "Open Sans"
  • accesskey attribute sets keyboard shortcuts for things like buttons and links!
